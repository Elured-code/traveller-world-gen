# Understanding `traveller_map_fetch.py`

A guide for Python beginners. This file fetches canonical world data from the public
TravellerMap API and builds a fully detailed system around it.

---

## What this file does

Instead of generating a world from scratch, this file:

1. **Fetches** the canonical UWP, stellar string, bases, trade codes, and PBG
   (Population-Belt-Gas digit) from the TravellerMap website
2. **Parses** the stellar classification string (e.g. `"G2 V M8 Ve"`) into
   `Star` objects that the orbit generator can use
3. **Generates** the orbital structure, secondary worlds, and moons procedurally
   from the canonical stellar data
4. **Overrides** the generated mainworld UWP with the canonical one, preserving
   the published data exactly

The result is a `TravellerSystem` that looks exactly like the published world but
has a fully realised physical and biological environment generated by the WBH rules.

| Section | What it contains |

## Key Python concept: HTTP requests

Python's standard library includes `urllib.request` for making HTTP requests:

```python
import urllib.request
import json

url = "https://travellermap.com/api/search?q=Regina&sector=Spinward+Marches"
with urllib.request.urlopen(url, timeout=10) as resp:
    data = json.loads(resp.read().decode())
```

`urllib.request.urlopen()` opens the URL like a file. `.read()` downloads the
response body as bytes. `.decode()` converts bytes to a Python string.
`json.loads()` parses the JSON string into Python dicts and lists.

The `with` statement ensures the connection is closed when the block ends —
even if an exception occurs.

---

## Key Python concept: regular expressions

The TravellerMap stellar string `"G2 V M8 Ve D"` needs to be parsed into individual
star descriptions. The file uses Python's `re` module (regular expressions):

```python
import re
stars = re.findall(r'[OBAFGKM]\d?\s+(?:Ia|Ib|II|III|IV|V|VI|D)?', stellar_string)
```

A regular expression is a pattern that matches text. `[OBAFGKM]` matches one
spectral type letter; `\d?` matches an optional digit; `\s+` matches whitespace;
`(?:...)` is a non-capturing group for the luminosity class alternatives. This is
more concise than a chain of `if` statements but requires practice to read fluently.

---

## Key Python concept: ambiguity handling

Many world names exist in multiple sectors (`"Regina"` appears in several). The file
detects this and raises a specific exception:

```python
class AmbiguousWorldError(Exception):
    def __init__(self, matches: list):
        self.matches = matches   # list of (world_name, sector_name) pairs
```

Raising a custom exception lets the caller (the GUI or the CLI) decide how to handle
the situation — by showing a disambiguation dialog, by asking the user to add a
sector, etc.

## The entry point

```python
system = generate_system_from_map(
    name="Regina",
    sector="Spinward Marches",
    seed=42,
    attach=True,          # run attach_detail() after generation
    nhz_atmospheres=True, # allow NHZ atmospheres for secondary worlds
)
```

Parameters:
- `name` + `sector` — identify the world; sector is always required
- `hex_pos` — alternative to name; the 4-digit hex coordinate in the sector grid
- `seed` — controls the procedural generation; same seed = same orbits and secondary worlds
- `attach` — if `True`, runs `attach_detail()` before returning
- `nhz_atmospheres` — stored in `TravellerSystem.nhz_atmospheres` for deterministic
  recreation; does not re-roll the canonical mainworld atmosphere (fixed UWP)
- `orbital_eccentricity`, `orbital_inclination` — gate eccentricity/inclination rolls

Returns a `TravellerSystem` with the canonical mainworld UWP but procedurally
generated everything else.

---

## Data provenance

The canonical UWP, stellar string, bases, trade codes, and travel zone come from
the TravellerMap API, which sources them from the Second Survey data set. These are
copyright the respective Traveller publishers. The procedurally generated orbital
structure, secondary world profiles, and moon data are produced by this project's
own generation modules.

## Running directly

```bash
python traveller_map_fetch.py --name Regina --sector "Spinward Marches"
python traveller_map_fetch.py --sector "Spinward Marches" --hex 1910 --seed 42 --detail
python traveller_map_fetch.py --name Mora --sector "Spinward Marches" --format json
```

`main()` handles argument parsing, calls `generate_system_from_map()`, and prints
the result in the requested format (text summary, JSON, or HTML).
