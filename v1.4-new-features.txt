Traveller World & System Generator — What's New in v1.4.0
==========================================================

GEN-UI — DARK MODE
-------------------
The app now has a dark mode. Toggle it via View > Dark Mode in the menu bar. The
setting is saved and restored the next time the app starts.

- Native Qt controls (window background, group boxes, labels, and badges) switch
  to a dark palette.
- The world and system HTML cards also switch to dark colours, matching the app
  rather than following the operating system setting.
- Toggling while a result is displayed updates the card immediately — no need to
  regenerate.

BIOLOGIC ATMOSPHERE TAINT (WBH p.83)
--------------------------------------
- Tainted atmospheres (codes 2, 4, 7, 9) can now produce a Biologic taint
  subtype. This taint appears in the atmosphere detail card as a taint with
  subtype "Biologic" (code B), representing alien micro-organisms or spores
  suspended in the air.

- When a biologic taint is present, the world is guaranteed to have at least
  Biomass 1 — life must exist to produce the taint. This enforcement happens
  automatically and is consistent with WBH p.83.

- Previously, biologic taint rolls were silently rerolled because the native
  life system (biomass ratings) was not yet implemented. Now that biomass
  generation is in place, biologic taints are fully enabled.

BELT MEAN TEMPERATURE (NEW IN WORLD CARD)
------------------------------------------
- Asteroid belt mainworlds (Size 0) now display a Mean temperature in Kelvin
  in the World Body detail card. The temperature is computed using the same
  WBH p.47 formula as for planetary worlds, with atmosphere DM set to zero
  (belts have no retained atmosphere).

GEN-UI — SYSTEM DETAIL CONTROLS REDESIGN
-----------------------------------------
- The flat row of radio buttons and checkboxes for system-detail sub-options
  has been replaced by a "System detail" group box (QGroupBox). Enabling the
  group-box checkbox activates the group; disabling it automatically grays out
  all four child options (NHZ Atmospheres, Oxygen requires biomass, Advanced
  temperature, and Runaway greenhouse). The dependency between System detail
  and its sub-options is now immediately visible without having to try each
  option to discover which ones are linked.

GEN-UI — TRAVELLERMAP PANEL SHOW/HIDE
----------------------------------------
- When "Procedural" mode is selected, the Sector, Name, and Hex fields are now
  hidden entirely rather than displayed grayed out. They reappear when
  "TravellerMap" is selected. This removes visual clutter and eliminates the
  confusion of visible-but-inactive fields.

GEN-UI — TRAVELLERMAP LOOKUP FEEDBACK
----------------------------------------
- TravellerMap lookups now run on a background thread (QThread). As soon as
  Generate is clicked, the status line shows "Looking up <world> in <sector>…"
  and the Generate button is disabled until the response arrives. The UI
  remains responsive and interactive during the lookup. Previously the app
  would freeze for the duration of a slow or timing-out TravellerMap request.

ORBIT TABLE — HORIZONTAL SCROLL
---------------------------------
- The system orbit survey table (11 columns: UWP, Name, Orbit#, Ecc/Incl,
  Period, Notes, and others) now scrolls horizontally when the panel is
  narrower than its content. Column headers no longer wrap to multiple lines.
  This is particularly noticeable on narrower window sizes or when the Notes
  column contains long text.

STANDALONE BINARIES — SMALLER FILE SIZE
-----------------------------------------
- v1.4.0 binaries are substantially smaller than v1.3.x through a combination
  of:
    • 37 unused Qt modules excluded from the bundle
    • Qt translation files removed (100+ languages not needed)
    • Unused image format plugins removed (only SVG, JPEG, GIF, ICNS, ICO
      retained)
    • Python bytecode optimised (docstrings and assertions stripped)
    • Debug symbols stripped on Linux and macOS
    • UPX compression enabled on all platforms

- macOS builds are now provided as separate architecture-native binaries
  (Apple Silicon arm64 only from v1.4.0 onwards). The Intel x86_64 build has
  been discontinued.

OPEN SYSTEM JSON — SECONDARY WORLD DETAIL NOW RESTORED
--------------------------------------------------------
- When a system JSON is opened (File > Open JSON…), secondary world profiles
  are now fully restored from the saved file. Previously only the main system
  structure (stars, orbit slots, mainworld) was reconstructed; the detail block
  for each secondary world slot (SAH, social data, moons, moon orbits, belt/world
  physical) was silently discarded.

- The system card now shows complete secondary world information — the same
  content that appears immediately after using "Full detail" generation — when
  a file saved with full detail is reopened.

SYSTEM MAP — WORLD MARKER POSITION
------------------------------------
- World glyphs on the orbit arc diagram now appear one third of the way down
  the arc from the top endpoint, rather than at the very top. Markers no longer
  cluster at the tip of short arcs and sit closer to the visual centre of the
  diagram, making multi-world systems easier to read at a glance.

HABITABILITY RATING (WBH p.131)
---------------------------------
- Every mainworld now receives a Habitability Rating — a score from 0 to 10+
  that indicates how suitable the world is for unprotected human habitation
  without survival gear.

- The rating is deterministic (no dice roll): it starts at 10 and applies
  modifiers for world size, atmosphere code, hydrographics, solar tidal lock,
  surface temperature, and surface gravity.

- Temperature modifiers use the physics-based advanced temperature figures when
  Advanced temperature is enabled; they fall back to the basic temperature
  category (Frozen, Cold, Hot, Boiling) otherwise.

- The rating appears in a new Habitability section of the world card, with a
  descriptive label:

    0         Actively hostile world
    1–2       Barely habitable world
    3–5       Marginally survivable world
    6–7       Regionally habitable world
    8–9       Suitable for human habitation
    10 or higher  Terra-equivalent garden world

- Terra itself scores exactly 10. A world with a thin oxygen atmosphere, mild
  temperature, and Earthlike gravity might score 8 or 9.

SYSTEM CARD — HABITABLE ZONE AND MINIMUM ARMISTICE ORBIT COLUMNS
------------------------------------------------------------------
- The Stars table in the System tab now shows two additional columns for each
  star: MAO (Minimum Armistice Orbit) and HZ Orbit# range.
- MAO is the minimum safe orbit for a treaty world — roughly one-third of the
  distance to the next star. It is shown as an Orbit# value to 2 decimal places.
  The primary star has no MAO and shows a dash.
- HZ Orbit# shows the inner edge, centre, and outer edge of the star's Habitable
  Zone as three Orbit# values separated by dashes, e.g. "4.50 – 5.00 – 5.50".
  Stars without a computed Habitable Zone show a dash.

SYSTEM TAB — MAINWORLD DETAIL REMOVED
--------------------------------------
- The System tab no longer duplicates the mainworld's UWP statistics,
  atmosphere, hydrographic, biological, habitability, and notes sections.
  Those details appear only on the Mainworld tab, which shows them via the
  full world card. The System tab now shows stellar data and the orbital
  survey table only.

SEISMIC HEATING — ABSORBED INTO TEMPERATURE DISPLAY
------------------------------------------------------
- The former "Seismic temperature" row in the world card has been removed.
  Tidal/seismic heating is now folded directly into the Advanced mean
  temperature, High temperature, and Low temperature values shown on the card.
  If seismic heating raises any of those figures, the new higher value is what
  appears — there is no longer a separate "what if you add seismic heating"
  display field.

GAS GIANT MASS — WBH TABLE NOW USED
-------------------------------------
- Gas giant mass is now rolled using the correct WBH three-tier mass table,
  rather than being estimated from the gas giant's size category:

    Small gas giant (GS):  10–35 Earth masses    (5 × 1D+1)
    Medium gas giant (GM): 40–340 Earth masses   (20 × 3D−1)
    Large gas giant (GL):  350–3,300 Earth masses (D3×50 × 3D+4)

- This rolled mass is stored in the system JSON and used everywhere a gas giant
  mass is needed: moon generation, tidal stress calculations, and Roche limit
  checks. A fallback based on diameter is used when loading older JSON files
  that predate this change.

MOONS NEAR THE ROCHE LIMIT NOW CONVERTED TO RINGS
---------------------------------------------------
- If a significant moon's closest approach to its parent gas giant (accounting
  for orbital eccentricity) falls inside the Roche limit (2.0 planetary diameters),
  it is tidally disrupted and converted to ring material. The ring count on the
  gas giant increases by one. This check runs after eccentricity rolls, so only
  moons with genuinely unsafe orbits are disrupted.

BUG FIXES
---------
- Fixed: Worlds orbiting a gas giant as a satellite displayed seismic stress
  values millions of times larger than physically possible. The tidal stress
  formula (calibrated for AU-scale star-to-planet distances) was incorrectly
  applied at moon distances (~0.1–0.8 Mkm), amplifying results by a factor of
  10¹⁵ or more. The formula is no longer applied at moon distances; only the
  tidal amplitude contribution remains. Additionally, the Tidal Stress Factor
  is now capped at 500 to prevent overflow at the display layer.

- Fixed: Cold-temperature worlds with standard breathable atmospheres (vacuum,
  trace, thin, standard, dense, etc.) were incorrectly shown as having an
  Ammonia ocean instead of Water. Ammonia oceans only form in chemically exotic
  environments — worlds with exotic, corrosive, insidious, or unusual atmospheres
  (codes A–F). All other Cold worlds now correctly display Water as the surface fluid.

- Fixed: `has_gas_giant` flag was incorrectly re-derived from `gas_giant_count`
  when loading a world JSON. If the two values ever differed (e.g., a world
  generated before the count was tracked, or an edited JSON), the flag would
  be wrong. The saved boolean is now read directly; `gas_giant_count` is used
  only as a fallback for legacy files that predate the explicit field.

- Fixed: worlds at extreme orbital distances (very cold, near absolute zero)
  could display a mean temperature of 3 K due to unchecked formula extrapolation.
  The WBH footnote specifies that results below 10 K should be replaced with a
  random 6–11 K roll. This is now correctly implemented.

- Fixed (Windows binary): the System tab and any web-view panel showed a blank
  white page. QtWebEngineProcess.exe and the WebEngine resource files (.pak,
  locales) were not included in the PyInstaller bundle when PySide6-Addons is
  installed separately from the base PySide6 package. These files are now
  explicitly collected in traveller_gen_ui.spec.

- Fixed (Windows install): a ModuleNotFoundError for PySide6.QtWebEngineWidgets
  at startup. PySide6-Addons (which provides QtWebEngineWidgets) was missing
  from gen-ui/requirements.txt. It is now listed with a compatible version
  constraint alongside PySide6.
