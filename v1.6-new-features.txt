Traveller World & System Generator — What's New in v1.6.0
==========================================================

RUNAWAY GREENHOUSE — NOW APPLIES TO SECONDARY WORLDS AND MOONS
-----------------------------------------------------------------
The optional Runaway Greenhouse check (World Builder's Handbook p.79) previously
only ever affected the mainworld. It now also applies to every eligible
secondary world and moon in the system — gas giants themselves are never
affected, but rocky and icy moons orbiting a gas giant are checked just like
any other secondary world or moon.

- There is no new option to turn on: this uses the same Runaway Greenhouse
  setting that already existed for the mainworld, in the desktop app, the web
  interface, and the API. Turning it on now affects the whole system instead
  of just the mainworld.
- The command-line tool gained a --runaway-greenhouse flag for the first time,
  so this check (mainworld and secondary worlds/moons alike) can now be used
  from the CLI as well as the desktop app and web API.

GEN-UI — EXPORT A3 POSTER
--------------------------
The desktop app can now export a printable A3 poster for a generated system.
After generating a full system with detail, choose File > Export A3 Poster…
to save it.

- Page 1 is a single landscape sheet with the perspective system map filling
  the whole page, and floating cards showing the system title, star list,
  mainworld's key stats, and a handful of notable bodies (gas giants and
  inhabited secondary worlds).
- Page 2 is the complete system card — full stars table and orbital survey
  table with every secondary world and moon — sized to match page 1.
- The save dialog offers two formats: HTML (open it in any browser and print
  from there) or PDF (saved directly, ready to print, no browser step needed).

GEN-UI — FILE > NEW AND FILE > NEW WITH NEW SEED
--------------------------------------------------
Two new menu items make it faster to regenerate without touching the seed field
by hand.

- File > New (Ctrl+N / Cmd+N) regenerates using whatever seed is currently
  shown in the seed field, with your current options. Useful for re-running
  the exact same system after changing an option, without having to retype
  the seed.
- File > New with New Seed (Ctrl+Shift+N) regenerates with a brand new random
  seed, keeping your current options — the same as clicking Generate with an
  empty seed field, available as a menu item and shortcut.

SYSTEM DETAIL CARD — NEW PRIMARY COLUMN IN THE STARS TABLE
-------------------------------------------------------------
The Stars table (desktop app System tab, web app, and the A3 poster's full
system card) now has a Primary column between Desig and Class, showing which
star each star orbits — "--" for the primary star itself, and the parent
star's designation for every companion or secondary star.

SYSTEM DETAIL CARD — COMPANION AND SECONDARY STARS NOW LISTED IN THE ORBITAL
SURVEY TABLE
-----------------------------------------------------------------------------
Companion stars and secondary stars (any star other than the primary) now
appear as a row in the Orbital survey table, listed under their own parent
star and positioned by orbital distance among the worlds that orbit that
same star. Previously a secondary star with no worlds of its own — for
example when its own orbital exclusion zone left it no room to host any —
was entirely missing from this table, even though it clearly occupied a gap
in its parent star's own world sequence.

SYSTEM MAP — A SECONDARY STAR'S OWN COMPANION IS NOW SHOWN NEXT TO IT
------------------------------------------------------------------------
When a secondary star (for example "B") itself has its own companion star
(for example "Ba"), the system map now also draws that companion as a small
satellite next to "B"'s own marker — in addition to its existing, correct
position in "B"'s own section of the map. The companion's distance from its
parent and its orbital tilt are both scaled to sit clearly next to the
parent star it belongs to.

NEW DEFAULT NAMING SCHEME FOR STARS, WORLDS, AND MOONS
-----------------------------------------------------------
Placeholder names generated for a system's stars, worlds, and moons follow a
new, more systematic pattern.

- Stars are named "<system name> <letter>", for example "Unknown A",
  "Unknown B", "Unknown Ba" — every star gets a name now, including
  companion stars, which previously had none.
- The system's main world is named "<system name> Prime" when you don't type
  a system name — for example "Unknown Prime". If you do type a name, the
  main world uses exactly that name, with no "Prime" added.
- Every other world (including asteroid belts, which previously used a
  separate naming pattern) is named "<system name> <star letter>-<number>",
  numbered in order of distance from its star — for example "Unknown A-1",
  "Unknown A-2".
- Moons are named after their world with a phonetic letter added — for
  example "Unknown A-1 ay", "Unknown A-1 bee" — instead of the previous
  Greek-letter names (alpha, beta, ...).

BUG FIXES
---------
- Fixed: the System Map window did not remember your Light/Dark theme or
  Perspective/Top-down view choice. Every newly opened map window reset to
  Dark theme and Top-down view regardless of what you had last selected. Both
  choices are now remembered and applied the next time you open a map window.
- Fixed: in the system map, a companion of a secondary star (for example
  "Ba", which orbits secondary star "B") was drawn right next to the
  primary star instead of next to the star it actually orbits.
- Fixed: in the Orbital survey table, the "#" numbering for a star's own
  worlds could incorrectly restart partway through the list instead of
  counting up continuously, in systems where a companion star's exclusion
  zone splits a star's worlds into two groups. In rare cases this could
  also cause a secondary world's generated details to be silently swapped
  with a different world's.
